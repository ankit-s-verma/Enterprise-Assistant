# Enterprise Assistant API Documentation

## Overview

The Enterprise Assistant exposes a RESTful API built with **FastAPI**. The API provides secure authentication using JWT tokens and allows authenticated users to interact with the Enterprise AI Assistant, employee directory, and ticket management system.

Interactive API documentation is automatically generated by FastAPI and is available at:

```text
http://localhost:8000/docs
```

---

# Base URL

Local Development

```text
http://localhost:8000
```

---

# Authentication

The API uses **JWT (JSON Web Token)** authentication.

All protected endpoints require an access token.

Example:

```http
Authorization: Bearer <access_token>
```

The token is obtained through the Login endpoint.

---

```text

      ▼
```

# API Endpoints

No authentication required.

### Response

```json

```

---

## User Login

### POST /login

Authenticates a user and returns a JWT access token.

### Authentication

Not required.

### Request

```json
{
    "username": "admin",
    "password": "password"
}
```

### Successful Response

```json
{
    "access_token": "<JWT_TOKEN>",
    "token_type": "bearer"
}
```

### Common Responses

| Status Code | Description                  |
| ----------- | ---------------------------- |
| 200         | Login successful             |
| 401         | Invalid username or password |
| 422         | Invalid request body         |

## AI Assistant

### POST /ask

Processes a user request using the Enterprise Assistant.

Depending on the user's intent, the assistant may:

* Answer company policy questions using Retrieval-Augmented Generation (RAG)
* Retrieve employee information
* Create IT support tickets
* Retrieve existing tickets
* Continue previous conversations using conversation memory

### Authentication

JWT Required

### Headers

```http
Authorization: Bearer <JWT_TOKEN>
Content-Type: application/json
```

### Example Request

```json
{
    "question": "Create a ticket for Outlook not opening.",
    "conversation_id": null
}
```

### Example Response

```json
{
    "answer": "Your IT support ticket has been created successfully.",
    "conversation_id": "1"
}
```

### Possible Operations

The assistant dynamically determines the user's intent and performs one or more of the following:

* Employee Lookup
* Ticket Creation
* Ticket Retrieval
* Knowledge Base Search
* Conversation Continuation
* General Question Answering

# Conversation Memory

The Enterprise Assistant maintains conversational context using a unique `conversation_id`.

For a new conversation:

```json
{
    "question": "Who is John Doe?",
    "conversation_id": null
}
```

The API returns a newly generated conversation ID.

Subsequent requests should include the same identifier:

```json
{
    "question": "What department does he work in?",
    "conversation_id": "4"
}
```

This enables context-aware interactions without requiring the user to repeat previous information.

# Retrieval-Augmented Generation (RAG)

The Enterprise Assistant uses a Retrieval-Augmented Generation pipeline to answer organization-specific questions.

Workflow:

1. User submits a question.
2. Relevant company documents are retrieved from ChromaDB.
3. Context is supplied to the Groq LLM.
4. The generated response is returned to the user.

This ensures responses are grounded in the organization's knowledge base rather than relying solely on the language model.

# Security

The API follows several security best practices:

* JWT-based authentication
* Password hashing using bcrypt
* Environment-based secret management
* Parameterized SQLAlchemy queries
* Input validation using Pydantic models
* Centralized exception handling
